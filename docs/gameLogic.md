# AffixForge — 遊戲核心邏輯與經濟模型說明

本插件為 Minecraft 引入了「槽位制（Slot-based）」裝備鍛造與附魔系統。裝備不再使用原版獨立的附魔機制，而是透過「槽位」共同爭奪裝備的潛能。

## 1. 鑑定機制 (Authentication System)
*   **初始狀態：** 所有合成、掉落或自定義的武器、防具、飾物（Accessory）預設為「未鑑定」。此時可正常使用原版附魔。
*   **鑑定消耗：** 玩家必須在專用 GUI 放入裝備，並消耗 **青金石磚 (Lapis Lazuli Block) x1** 進行鑑定。
*   **鑑定結果：**
    1.  裝備轉化為「槽位制」，原版附魔轉為停用（若有附魔，需觸發下方轉移書邏輯）。
    2.  根據機率決定物品的 **品質（5階）**，並賦予初始詞條與對應的最大槽位上限：
        *   **普通 (Common)：** 1~2 槽位 (白色)
        *   **罕見 (Uncommon)：** 2~3 槽位 (綠色)
        *   **稀有 (Rare)：** 3~4 槽位 (藍色)
        *   **史詩 (Epic)：** 4~5 槽位 (紫色)
        *   **傳說 (Legendary)：** 5 槽位 (金色)
*   **重複鑑定：** 玩家可重複消耗青金石磚進行鑑定，重新刷新品質、初始詞條與初始槽位。

## 2. 槽位消耗與玉髓突破 (Slot Mechanics)
*   **槽位互斥：** 「原版附魔」與「自定義詞條」共同消耗裝備的總槽位。每個附魔或詞條預設消耗 **1 個槽位**（不論等級，或依未來配置調整）。
*   **玉髓打破上限：** 玩家可消耗 **玉髓錠 (Chalcedony Ingot) x1**，強制為已鑑定裝備增加 1 個最大槽位上限。每把裝備最多可擴充至 **16 個槽位**。

## 3. 洗練、鎖定與升級 (Reforging & Upgrading)
在鍛造 GUI 中，玩家可以對已鑑定裝備進行改造：
*   **隨機洗練：** 消耗青金石，隨機重置裝備「未鎖定」的詞條內容與其初始等級。
*   **戰略鎖定：** 消耗 **鑽石磚** 鎖定極品詞條不被洗掉（鎖定 1/2/3/4 條，分別消耗 1/2/4/8 個鑽石磚）。第 5 條後無法鎖定。
*   **等級升級：** 玩家可以單獨消耗鑽石磚，指定提升某個已有詞條的等級（例如將其從 Level I 升級至 Level II），直到該詞條的 `max_level`。

## 4. 萬用附魔轉移書 (Enchantment Extraction)
*   **機制：** 玩家可放入「已附魔裝備」+「普通書本x1」，完整將裝備上的原版附魔剝離，武器變回無附魔狀態，並獲得一張對應的「附魔書」。
*   **時機：** 用於鑑定已有附魔的武器，或作為伺服器日常的通用功能。
